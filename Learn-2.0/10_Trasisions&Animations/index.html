Introduction

CSS transitions and animations allow you to create engaging, interactive user interfaces that respond smoothly to user actions. They help improve user experience by providing visual feedback and guiding users through the interface.

In this documentation, we'll cover:

  • CSS transitions for smooth property changes
  • CSS animations with keyframes for complex movement
  • Hover effects to enhance interactivity
  • Common UI interactions like dropdowns, modals, and accordions
  • Best practices for performance and accessibility

CSS Transitions

CSS transitions allow property changes in CSS values to occur smoothly over a specified duration, rather than happening instantly. They're perfect for simple state changes, like hover effects.

Transition Properties

The main transition properties are:

  • transition-property: Specifies which CSS property to animate
  • transition-duration: How long the transition should take (in seconds or milliseconds)
  • transition-timing-function: The speed curve of the transition
  • transition-delay: Optional delay before the transition starts
  • transition: Shorthand property for all of the above

Basic Transition Example

/* CSS */
.element {
    width: 100px;
    height: 100px;
    background-color: blue;
    /* Apply transition to all properties */
    transition: all 0.3s ease;
}

.element:hover {
    background-color: red;
    transform: scale(1.2);
}

Timing Functions

Timing functions control how the intermediate values of the properties being affected by a transition are calculated:

  • ease: Starts slow, speeds up, then ends slowly (default)
  • ease-in: Starts slow, ends fast
  • ease-out: Starts fast, ends slow
  • ease-in-out: Starts and ends slow
  • linear: Same speed from start to end
  • cubic-bezier(n,n,n,n): Custom timing function

Multiple Transitions Example

/* CSS */
.element {
    width: 100px;
    height: 100px;
    background-color: blue;
    /* Apply different transitions to different properties */
    transition: background-color 0.5s ease,
                transform 0.3s ease-in-out,
                border-radius 0.4s linear;
}

.element:hover {
    background-color: purple;
    transform: rotate(45deg);
    border-radius: 50%;
}

Which Properties Can Be Transitioned?

Most CSS properties can be transitioned, but some perform better than others. Properties that work well include:

  • Colors (color, background-color, etc.)
  • Dimensions (width, height, padding, margin)
  • Transformations (transform properties)
  • Opacity and visibility
  • Border properties

Properties that can't be transitioned include: display, position, and font-family.

CSS Animations

While transitions are ideal for simple state changes, CSS animations allow more complex, multi-step animations that can run automatically without requiring user interaction.

Animation Properties

The main animation properties include:

  • animation-name: Specifies the name of the @keyframes rule
  • animation-duration: How long the animation should take to complete one cycle
  • animation-timing-function: The speed curve of the animation
  • animation-delay: Optional delay before the animation starts
  • animation-iteration-count: How many times the animation should repeat
  • animation-direction: Whether the animation should play in reverse on alternate cycles
  • animation-fill-mode: Specifies how the element should be styled before/after the animation
  • animation-play-state: Specifies whether the animation is running or paused
  • animation: Shorthand property for all of the above

Basic Animation Example

/* CSS */
@keyframes bounce {
    0%, 100% {
        transform: translateY(0);
    }
    50% {
        transform: translateY(-30px);
    }
}

.element {
    width: 100px;
    height: 100px;
    background-color: blue;
    /* Apply animation */
    animation: bounce 2s infinite ease-in-out;
}

Animation Control

You can control animations with JavaScript by changing the animation-play-state property or by adding/removing classes with animations.

Animation Control Example

/* CSS */
@keyframes bounce {
    0%, 100% { transform: translateY(0); }
    50% { transform: translateY(-30px); }
}

.element {
    animation: bounce 2s infinite ease-in-out;
    animation-play-state: running; /* or paused */
}

/* JavaScript */
document.getElementById('pause-btn').addEventListener('click', function() {
    document.getElementById('animation-element').style.animationPlayState = 'paused';
});

document.getElementById('play-btn').addEventListener('click', function() {
    document.getElementById('animation-element').style.animationPlayState = 'running';
});

CSS Keyframes

The @keyframes rule is used to define the stages and styles of an animation sequence. It allows you to specify multiple points during an animation where you want specific styles to be applied.

Keyframe Syntax

@keyframes animationName {
    0% {
        /* CSS properties at start */
    }
    50% {
        /* CSS properties at middle */
    }
    100% {
        /* CSS properties at end */
    }
}

You can also use the keywords from (equivalent to 0%) and to (equivalent to 100%):

@keyframes animationName {
    from {
        /* CSS properties at start */
    }
    to {
        /* CSS properties at end */
    }
}

Complex Keyframe Example

/* CSS */
@keyframes colorChange {
    0% { background-color: #6e8efb; }
    25% { background-color: #a777e3; }
    50% { background-color: #f06292; }
    75% { background-color: #43a047; }
    100% { background-color: #6e8efb; }
}

.element {
    width: 100px;
    height: 100px;
    animation: colorChange 8s infinite;
}

Multiple Property Changes

Keyframes can animate multiple properties at once:

@keyframes move-and-color {
    0% {
        transform: translateX(0);
        background-color: blue;
        border-radius: 0;
    }
    50% {
        transform: translateX(100px);
        background-color: purple;
    }
    100% {
        transform: translateX(0);
        background-color: red;
        border-radius: 50%;
    }
}

}

Best Practices

To ensure your transitions and animations perform well and enhance the user experience, follow these best practices:

Performance Optimization

  • Use transforms and opacity: These properties are hardware-accelerated in most browsers, resulting in smoother animations.
  • Avoid animating expensive properties: Properties like box-shadow, filter, and width/height can cause layout recalculation and are more performance-intensive.
  • Use the will-change property: This hints to browsers which properties are expected to change, allowing for optimizations.
  • Keep animations short: Aim for 300ms or less for micro-interactions to maintain a snappy feel.
  • Use appropriate timing functions: Different easing functions create different feelings of movement.
/* Good for performance */
.element {
    transform: translateZ(0); /* Hardware acceleration hint */
    will-change: transform, opacity;
    transition: transform 0.3s ease, opacity 0.3s ease;
}

Accessibility Considerations

  • Respect user preferences: Use prefers-reduced-motion media query to disable or simplify animations for users who prefer reduced motion.
  • Avoid rapid flashing: Animations should not flash more than three times per second to prevent triggering photosensitive conditions.
  • Make sure interactive elements remain accessible: Animations shouldn't obscure important content or make interactive elements difficult to use.
/* Respecting user preferences */
@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration: 0.001ms !important;
        transition-duration: 0.001ms !important;
    }
}

Browser Compatibility

While CSS transitions and animations have excellent browser support, consider the following:

  • Use appropriate vendor prefixes or a tool like Autoprefixer for older browsers.
  • Test your animations across different browsers and devices.
  • Provide fallbacks for browsers that don't support certain features.
/* Vendor prefixes example */
.element {
    -webkit-animation: bounce 2s;
    animation: bounce 2s;
}

@-webkit-keyframes bounce {
    /* keyframes */
}
@keyframes bounce {
    /* keyframes */
}

Advanced Animation Techniques

GSAP Animation Library

For more complex animations, consider using a JavaScript animation library like GSAP (GreenSock Animation Platform), which provides powerful tools for creating sophisticated animations:

// GSAP example
gsap.to(".element", {
    duration: 1,
    x: 100,
    y: 50,
    rotation: 360,
    scale: 1.5,
    ease: "elastic.out(1, 0.3)"
});

Staggered Animations

Creating staggered animations, where multiple elements animate sequentially:

/* CSS approach */
.stagger-item:nth-child(1) { animation-delay: 0.1s; }
.stagger-item:nth-child(2) { animation-delay: 0.2s; }
.stagger-item:nth-child(3) { animation-delay: 0.3s; }

/* JavaScript approach (GSAP) */
gsap.to(".stagger-item", {
    duration: 0.5,
    opacity: 1,
    y: 0,
    stagger: 0.1
});

Scroll-Triggered Animations

Animations that trigger as elements scroll into view:

/* Using Intersection Observer API */
const observer = new IntersectionObserver((entries) => {
    entries.forEach(entry => {
        if (entry.isIntersecting) {
            entry.target.classList.add('animate');
        }
    });
}, { threshold: 0.1 });

document.querySelectorAll('.scroll-animate').forEach(element => {
    observer.observe(element);
});

Variable Animations with CSS Custom Properties

Using CSS variables to create dynamic animations:

:root {
    --animation-speed: 1s;
    --animation-distance: 100px;
}

.element {
    animation: slide var(--animation-speed) ease;
}

@keyframes slide {
    to { transform: translateX(var(--animation-distance)); }
}

/* Change variables with JavaScript */
document.querySelector('.element').style.setProperty('--animation-distance', '200px');

Troubleshooting Common Issues

Animation Not Working

  • Check for typos in property names or keyframe names
  • Verify that the element is displayed (animations don't work on display: none elements)
  • Make sure animation properties have the correct syntax
  • Inspect using browser developer tools to see applied styles

Choppy Animations

  • Use transform and opacity instead of other properties
  • Add will-change property
  • Reduce the complexity of animations
  • Consider using requestAnimationFrame for JavaScript animations

Unexpected Animation Behavior

  • Check animation-fill-mode property
  • Verify that keyframes are properly defined
  • Ensure animation duration is appropriate
  • Check for conflicting transitions or animations
